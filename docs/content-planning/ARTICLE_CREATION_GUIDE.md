# Инструкция по созданию статей MargoFitness

## Контекст проекта

MargoFitness — онлайн-платформа персональных домашних тренировок для женщин 25–45 лет (начинающие и средний уровень). Гибридная бизнес-модель: подписки (free / basic / pro) + разовые покупки.

Статьи — это полноценный контент-продукт: они обучают, вовлекают, формируют доверие к бренду и мотивируют пользоваться платформой.

---

## Тон и стиль (Tone of Voice)

- **Автор**: Марго — основатель, эксперт, подруга. Не коуч-мотиватор, не врач-менторша.
- **Обращение**: на «ты», по-дружески, но с экспертностью.
- **Язык**: профессиональный, но доступный. Научные данные объясняются просто.
- **Позиция**: поддержка без давления. Никакого body-shaming, никакого «соберись, тряпка».
- **Ценности**: сила, забота о себе, прогресс (не перфекционизм), доступность.
- **Запрещено**: агрессивный маркетинг, кликбейт, необоснованные обещания, «похудей за 7 дней».

## Техническая реализация

### Формат

Статьи — это **хардкодные React-компоненты** (НЕ MDX, НЕ Markdown). Каждая статья — отдельный `.tsx` файл.

### Расположение файлов

- Компонент статьи: `components/articles/content/{ArticleName}.tsx`
- Реестр метаданных: `lib/config/articles.ts` → массив `ARTICLE_REGISTRY`
- Динамический импорт: `app/dashboard/health-tracker/components/workouts-tab.tsx` → объект `HardcodedArticles`

### Регистрация новой статьи (3 шага)

**1. Создать компонент** в `components/articles/content/{ArticleName}.tsx`

**2. Добавить метаданные** в `lib/config/articles.ts`:
```typescript
{
  id: "slug-name",
  slug: "slug-name",
  title: "Заголовок статьи",
  description: "Краткое описание для карточки в списке (1-2 предложения).",
  category: "Основы" | "Тренировки" | "Питание" | "Биохакинг",
  reading_time: 7,  // минут
  access_level: "free" | "basic" | "pro",
  image_url: "https://images.unsplash.com/..."  // для карточки в списке
}
```

**3. Добавить динамический импорт** в `workouts-tab.tsx`:
```typescript
"slug-name": dynamic(() => import("@/components/articles/content/ArticleName")),
```

### Обязательная структура компонента

```typescript

import React from "react";

import { useArticleReadTracking } from "@/app/dashboard/health-tracker/hooks/use-article-read-tracking";
import { markArticleAsRead } from "@/lib/actions/articles";

export default function ArticleName({
  onBack,
  metadata,
}: {
  onBack: () => void;
  metadata?: any;
}) {
  const { elementRef } = useArticleReadTracking({
    articleId: metadata?.id || "slug-name",
    onRead: async (id) => {
      await markArticleAsRead(id);
    },
    threshold: 0.5,
  });

  return (
    <motion.div
      initial={{ opacity: 0, y: 20 }}
      animate={{ opacity: 1, y: 0 }}
      className="text-white selection:bg-{accentColor}-500/30"
    >
      {/* HERO */}
      {/* ... */}

      {/* ARTICLE BODY */}
      <article className="max-w-[860px] mx-auto px-4 md:px-0">
        {/* Контент статьи */}
        {/* ... */}

        {/* CTA секция */}
        {/* ... */}

        {/* Трекер прочтения — ОБЯЗАТЕЛЬНО в конце */}
        <div ref={elementRef} className="h-4 w-full" />
      </article>
    </motion.div>
  );
}
```

## Объём и глубина статьи

### Ориентиры

| Параметр | Рекомендуемый диапазон |
|----------|----------------------|
| Текст (без кода) | 1 000 – 1 500 слов |
| Основные секции | 4 – 6 штук |
| Время чтения | 6 – 9 минут |
| Строки кода компонента | 500 – 900 строк |

### Эталонные статьи по объёму

**SafetyBasics** (~676 строк, 6 мин чтения) и **NutritionBasics** (~900 строк, 7 мин чтения) — оптимальный объём. Текст в них плотный, каждая секция раскрыта, визуальные элементы оправданы.

**HomeVsGym** (~353 строки, 6 мин чтения) — минимально допустимый объём. Подходит для узкой, конкретной темы.

### Принцип глубины

Статья не должна быть поверхностной. Если тема позволяет раскрыть несколько аспектов — раскрывай. Лучше написать 5 глубоких секций, чем 3 поверхностных. Признаки недостаточной глубины:
- Секция содержит только 1–2 абзаца без визуальных элементов
- Тезисы не подкреплены примерами, данными или объяснениями «почему»
- Статья заканчивается раньше, чем читатель получает практическую ценность
- Есть аспекты темы, которые очевидно подходят для раскрытия, но пропущены

Не раздувай текст искусственно, но и не экономь на объяснениях. Каждое утверждение должно быть либо объяснено (почему это работает), либо проиллюстрировано (пример, аналогия), либо подкреплено (данные, исследование).

## Анатомия статьи (обязательные блоки)

### 1. Hero-секция

Полноширинная шапка с двухколоночной раскладкой (текст + изображение).

**Обязательные элементы:**
- Кнопка «Назад к материалам» (desktop, `hidden md:block`)
- Бейдж категории (цвет категории, text-black)
- Бейдж access_level (опционально, `hidden md:inline-block`)
- Бейдж времени чтения с иконкой Clock
- Заголовок H1: `font-oswald font-black uppercase tracking-tighter leading-[0.95]`
  - Размеры: `text-3xl md:text-6xl`
  - Акцентная часть заголовка: `text-{accentColor}-400`
- Подзаголовок: italic, `border-l-2 border-{accentColor}-400/30 pl-8`
- Изображение: `grayscale opacity-60` с градиентным оверлеем

**Паттерн Hero (копировать):**
```
rounded-[2rem] md:rounded-[3rem] border border-white/10 bg-white/[0.02] mb-16
  → grid grid-cols-1 lg:grid-cols-2 min-h-[400px] md:min-h-[500px]
    → Левая колонка: p-8 md:p-16 pt-6 md:pt-24
    → Правая колонка: h-64 lg:h-auto overflow-hidden + gradient overlays
```

### 2. Вступление

2 абзаца после hero. Задаёт проблему и обещает решение. Без заголовка.
- `text-lg md:text-xl text-white/70 leading-relaxed`
- Контейнер: `space-y-6 mb-14 text-left`

### 3. Основные секции (контент)

Каждая секция — это `<section className="mb-14 text-left">` с:
- Заголовком секции (H2): `text-2xl md:text-3xl font-oswald font-black uppercase tracking-tight text-white`
- Текстом: `text-lg text-white/70 leading-relaxed`
- Визуальными элементами (инфографика, карточки, таблицы)

### 4. Блок платформы (обязательный)

В каждой статье должен быть рекламный блок, который связывает тему статьи с функционалом MargoFitness. Это НЕ баннер и НЕ прямая реклама — это органичная секция, которая показывает, как платформа помогает решить проблему, о которой идёт речь в статье.

**Правила:**
- Дизайн блока уникален для каждой статьи — нельзя копировать один и тот же шаблон
- Информация должна соответствовать реальному функционалу платформы (см. список ниже)
- Блок должен логически вытекать из контента статьи
- Тон: «вот как мы это решаем», а не «купи подписку»
- Размещение: ближе к середине или концу статьи, но ДО финального CTA

**Примеры реализации в существующих статьях:**
- **SafetyBasics** — «Твоя безопасность — наш приоритет»: 2-колоночный блок (текст + мокап интерфейса), акцент на разборе техники и видео-инструкциях
- **HabitMagic** — «Платформа, которая строит привычки, а не наказывает за пропуски»: 3 карточки (трекер привычек, сообщество, прогресс без давления)
- **EquipmentGuide** — «Инвентарь готов? Пора в дело»: превью тренировки с характеристиками + кнопка перехода

#### Реальный функционал платформы (только из этого списка)

**Тренировки:**
- Еженедельные программы тренировок (2–3 сессии в неделю)
- Видео-инструкции к каждому упражнению (Kinescope)
- Детальное текстовое описание техники с ключевыми точками контроля
- Таймеры отдыха между подходами
- Библиотека упражнений с категориями и описанием инвентаря
- Отслеживание завершения тренировок с оценками (удовлетворённость + сложность)
- Демо-тренировки для бесплатного тарифа

**Трекер здоровья (виджеты):**
- Вода (дневное потребление с целью)
- Шаги (с целью)
- Вес (с ИМТ, трендами, целевым весом)
- Сон (часы с целью)
- Кофеин
- Настроение (шкала 1–5) + энергия (1–10)
- Питание (калории с целью, качество питания)
- Фото прогресса (фронт/бок/спина, понедельное)
- Замеры тела (грудь, талия, бёдра)

**Трекер привычек:**
- Создание и управление привычками
- Категории: утро, день, вечер, в любое время
- Отслеживание серий (текущая и максимальная)
- Тепловая карта привычек
- Статистика выполнения

**Статистика и аналитика:**
- Дашборд статистики с выбором периода (неделя/месяц/год)
- Графики и визуализации по каждой метрике
- Сравнение фото (до/после с разницей в весе)
- Тренды и средние значения

**Достижения:**
- Система достижений (обычные, редкие, эпические, легендарные)
- Автоматическая разблокировка при выполнении условий
- Достижения за серии, тренировки, привычки, метрики

**Сообщество:**
- Telegram-сообщество (до 2000 участников)
- Доступно на всех тарифах включая бесплатный

**Бонусная система:**
- Бонусные баллы (шаги) как валюта
- Кэшбек от покупок (3–10% по уровням)
- Реферальная программа

**Подписки:**
- Free: демо-тренировки, 1 виджет, 1 привычка
- Basic (299₽/мес): 2 тренировки/нед, 6 виджетов, 6 привычек, статистика
- Pro (599₽/мес): 3 тренировки/нед, все 8 виджетов, 10 привычек
- Elite (999₽/мес): персональное сопровождение, индивидуальный план питания, прямой контакт с Марго

**НЕЛЬЗЯ упоминать в рекламных блоках:**
- Функции, которых нет в списке выше
- Конкретные цены (они могут измениться)
- Обещания конкретных результатов («похудеешь на X кг»)
- Сравнение с конкурентами

### 5. CTA-секция (финал)

Последняя секция статьи. Завершает на сильной ноте — без пересказа и «блока резюме». Не надо суммировать прочитанное в блоке «Главное» — это выглядит слабо. Вместо этого: мощный CTA, который эмоционально закрывает статью и направляет к действию.

- `text-center py-8 md:py-12 border-t border-white/5 mt-8 md:mt-20 pb-32`
- Заголовок: `text-4xl md:text-6xl font-oswald font-black uppercase tracking-tighter`
- Подзаголовок: `text-white/40 text-lg md:text-xl max-w-xl mx-auto`
- Кнопка: `rounded-full bg-{accentColor}-500 hover:bg-{accentColor}-600 font-bold text-sm uppercase tracking-widest shadow-xl shadow-{accentColor}-500/15`
- Текст кнопки: «Начать тренировку» + ArrowRight

Примеры хороших CTA-заголовков:
- «Начни сегодня. Не с понедельника.» (HabitMagic)
- «Инвентарь — это инвестиция в себя» (EquipmentGuide)
- «Готова начать?» (HomeVsGym)

### 6. Трекер прочтения

`<div ref={elementRef} className="h-4 w-full" />` — невидимый элемент в самом конце `<article>`. Когда пользователь доскроллит до него, статья отмечается как прочитанная.

## Визуальные элементы и инфографика

### Принцип: форма следует за содержанием

Какие именно визуальные элементы использовать — определяется темой статьи. Не существует обязательного набора. Точно так же цитаты от Марго, блоки «Совет от Марго», сравнительные таблицы — всё это инструменты, которые используются только если они усиливают конкретную статью.

Визуальный элемент оправдан, если он:
- Передаёт информацию, которую текст передаёт хуже (сравнение, структура, процесс)
- Помогает запомнить (чеклист, резюме)
- Создаёт интерактивность (калькулятор, аккордеон)
- Разбивает монотонный текст, создавая ритм чтения

Визуальный элемент НЕ оправдан, если:
- Он дублирует то, что уже сказано текстом
- Добавлен «для красоты» без информационной нагрузки
- Занимает место, но не даёт ценности читателю

### Доступные форматы (справочник)

Ниже — каталог элементов, которые уже использовались в статьях. Это не список обязательных компонентов, а палитра, из которой ты выбираешь подходящее. Можно придумывать новые форматы, если они лучше подходят теме.

### Типы элементов (использованные в статьях)

**Сравнительные таблицы** (HomeVsGym — зал vs дом):
- Grid 3 колонки: параметр / вариант A / вариант B
- Акцентный цвет для «выигрывающей» стороны

**Карточки-концепты** (HomeVsGym — 3 кита адаптации):
- Grid 1–3 колонки, каждая с иконкой, заголовком, описанием
- Разные акцентные цвета для визуального разделения

**Анимированные прогресс-бары** (HomeVsGym — минуты работы):
- `motion.div` с `whileInView` для анимации ширины
- Сравнение двух значений

**Интерактивный калькулятор** (NutritionBasics — калории):
- Поля ввода с мгновенным расчётом
- `useState` для данных, `useMemo` для вычислений
- Результаты с анимацией `AnimatePresence`

**Каскад/таймлайн** (NutritionBasics — последствия голодания):
- Вертикальная линия с точками разных цветов (severity)
- Шаги с нарастающей серьёзностью

**Сценарные карточки** (NutritionBasics — 6 сценариев):
- Grid 2 колонки, каждая с бейджами факторов (вкл/выкл) и рейтинг-баром
- Разные акцентные цвета по результату

**DO / DON'T карточки** (SafetyBasics):
- 2 колонки: зелёная (делай) + красная (не делай)
- Иконки CircleCheck / CircleX

**Пошаговый процесс** (SafetyBasics — разминка):
- Вертикальная линия с точками
- Время + заголовок + описание

**Двухколоночное сравнение** (SafetyBasics — выдох/вдох):
- Grid 2 колонки с визуальным контрастом

**Барчарты в тексте** (HabitMagic — мотивация vs привычка):
- Массив значений → div'ы с высотой в процентах
- Визуально показывают тренд (угасающий vs растущий)

**Анимированная инфографика** (HabitMagic — петля привычки):
- SVG-круг с анимированным `strokeDashoffset`
- 4 элемента по углам, подсвечиваются по мере движения точки
- Центральный элемент с пульсацией

**Горизонтальный слайдер** (HabitMagic — 5 стратегий):
- `overflow-x-auto` с snap-scroll
- Навигация точками + кнопки < >
- Drag-to-scroll на десктопе

**Фазы с прогресс-барами** (HabitMagic — 66 дней):
- 3 карточки с цветовой кодировкой (danger/warning/success)
- Прогресс-бар + описание фазы

**Аккордеон** (EquipmentGuide — оборудование):
- `useState` для open/close
- `AnimatePresence` + `motion.div` для плавного раскрытия
- Внутри: «как выбрать» + «используется для» + «чем заменить»

**Колонки приоритетов** (EquipmentGuide):
- 3 колонки с разной визуальной «весомостью»
- must = яркий, recommended = средний, optional = приглушённый

**Блок платформы** (HabitMagic, SafetyBasics):
- Градиентный фон `from-{accentColor}-500/[0.06] via-white/[0.02] to-transparent`
- Описание функций платформы в контексте темы статьи
- НЕ прямая реклама, а объяснение как платформа помогает решить проблему из статьи

**Цитата от Марго** (HomeVsGym, NutritionBasics):
- Вертикальная линия + иконка Quote + italic текст
- Аватар Марго: `/images/avatars/margo.png` в круглой рамке
- Используй, когда нужно подчеркнуть ключевую мысль от первого лица

**Блок «Совет от Марго»** (SafetyBasics, HabitMagic):
- Иконка Lightbulb + подпись uppercase
- Используй, когда есть конкретный практический совет, который стоит выделить из основного текста

## Правила уникальности

**Каждая статья должна иметь уникальную структуру.** Нельзя использовать один и тот же набор компонентов в двух статьях. Примеры:

| Статья | Уникальные элементы |
|--------|-------------------|
| HomeVsGym | Сравнительная таблица, прогресс-бары, концепт-карточки |
| NutritionBasics | Калькулятор калорий, каскад последствий, сценарные карточки с факторами |
| SafetyBasics | DO/DON'T карточки, пошаговая разминка, красные флаги, блок платформы |
| HabitMagic | Барчарты мотивация vs привычка, SVG-анимация петли, слайдер стратегий, фазы 66 дней |
| EquipmentGuide | Аккордеон оборудования, приоритетные колонки, таблица весов, поэтапный чеклист покупок |

## Стилевые константы (Tailwind)

### Типографика
- Заголовки: `font-oswald font-black uppercase tracking-tight text-white`
- Подзаголовки секций: `text-2xl md:text-3xl`
- Основной текст: `text-lg text-white/70 leading-relaxed`
- Мелкий текст / подписи: `text-sm text-white/40 leading-relaxed`
- Метки: `text-[10px] font-bold uppercase tracking-widest text-white/30`

### Форматирование текста (акценты)

В основном тексте статьи используются inline-акценты для выделения ключевых слов и понятий. Это делает текст живым, помогает сканированию и расставляет смысловые ударения.

**Цветовой акцент (ключевые понятия):**
```html
<span className="text-{accentColor}-400/85 font-bold">ключевое слово</span>
```
Используется для терминов, важных концепций, цифр. Цвет соответствует категории статьи (rose, sky, amber, cyan).

**Цветовой акцент + подчёркивание (максимальный акцент):**
```html
<span className="text-{accentColor}-400/85 font-bold underline decoration-{accentColor}-500/30 underline-offset-4">текст</span>
```
Для самых важных выводов, ключевых фраз. Подчёркивание полупрозрачное, с отступом.

**Белый акцент (усиление без цвета):**
```html
<span className="text-white/90 font-bold">текст</span>
```
или
```html
<span className="text-white/85 font-bold">текст</span>
```
Для выделения слов, которые важны, но не являются терминами.

**Белый акцент + подчёркивание:**
```html
<span className="text-white/85 font-bold underline decoration-white/10 underline-offset-4">текст</span>
```
Мягкое подчёркивание для логических ударений.

**Мягкий акцент (italic + semibold):**
```html
<span className="text-white/85 font-semibold italic">текст</span>
```
или
```html
<span className="text-{accentColor}-400/85 font-semibold italic underline decoration-{accentColor}-500/20 underline-offset-4">текст</span>
```
Для метафор, образных выражений, мягких акцентов.

**Inline bold (`<strong>`):**
```html
<strong className="text-white/80">слово</strong>
```
Для коротких выделений внутри списков и описаний.

**Правила использования акцентов:**
- В каждом абзаце 1-3 акцента, не больше. Перебор убивает эффект.
- Цветовые акценты - для терминов и ключевых понятий статьи.
- Белые акценты - для логических ударений и контрастов.
- Подчёркивание - спарингово, только для самых важных фраз (1-2 на секцию).
- Не выделяй предлоги, союзы и общие слова.

### Тире и пунктуация

- Используй **короткое тире** `-` вместо длинного тире `—` во всех текстах статей.
- Пример: «Это не мнение, это - физика» (НЕ «Это не мнение, это — физика»)

### Контейнеры
- Карточки: `rounded-2xl bg-white/[0.03] border border-white/10 p-6 md:p-8`
- Акцентные карточки: `rounded-2xl bg-{color}-500/[0.04] border border-{color}-500/12`
- Предупреждения: `rounded-2xl bg-amber-500/[0.04] border border-amber-500/12` или `bg-rose-500/[0.05] border border-rose-500/15`
- Иконки в кружках: `size-10 rounded-xl bg-{color}-500/10 border border-{color}-500/15 flex items-center justify-center`

### Фон и прозрачности
- Фон страницы: `#09090b` (через bg-[#09090b])
- Полупрозрачный белый: `bg-white/[0.02]`, `bg-white/[0.03]`
- Границы: `border-white/10`, `border-white/5`
- Текст: `text-white/90` (акцент bold), `text-white/85` (акцент мягкий), `text-white/80` (inline strong), `text-white/70` (основной), `text-white/40` (вторичный), `text-white/25` (приглушённый)

### Анимации
- Вход страницы: `initial={{ opacity: 0, y: 20 }} animate={{ opacity: 1, y: 0 }}`
- Прогресс-бары: `initial={{ width: 0 }} whileInView={{ width: "X%" }} viewport={{ once: true }}`
- Аккордеоны: `AnimatePresence` + `motion.div` с `height: 0 → auto`
